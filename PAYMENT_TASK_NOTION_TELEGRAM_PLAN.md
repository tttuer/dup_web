# 납부 요청 노션 캘린더·텔레그램 연동 계획

- 기록일: 2026-07-23
- 기준 시간대: `Asia/Seoul`
- 원본 데이터: DUP의 `PaymentTask`

## 1. 결정 사항

납부 업무의 기준 데이터는 DUP에 저장한다.

- **노션 캘린더**: 매일 확인하는 주 화면으로 사용한다.
- **텔레그램**: 아침에 납부 건수만 알려주는 보조 알림으로 사용한다.
- **DUP 내부 알림**: 현재 웹소켓·브라우저 알림을 그대로 유지한다.

노션과 텔레그램에는 계좌번호, 첨부파일, 상세 금액 등 민감한 정보를 복사하지 않는다. 상세 내용은 DUP에 로그인한 뒤 확인한다.

## 2. 전체 흐름

1. DUP에서 납부 요청을 생성한다.
2. 백엔드가 노션의 `DUP 납부 일정` 데이터베이스에 일정을 생성한다.
3. 사용자는 노션 캘린더에서 납부 일정을 확인한다.
4. 노션 일정 안의 `DUP에서 확인` 링크를 누른다.
5. DUP가 해당 납부 업무를 바로 열어 보여준다.
6. 매일 오전 텔레그램으로 오늘 납부 건수를 한 번 알려준다.

## 3. DUP 바로가기

납부 업무마다 다음 형식의 고정 주소를 사용한다.

```text
https://arc.baeksung.kr/approval/payment-tasks/{task_id}
```

링크를 열었을 때의 동작은 다음과 같다.

- 납부 업무 탭을 자동으로 선택한다.
- URL의 `task_id`에 해당하는 업무를 API로 조회한다.
- 해당 업무를 화면 상단에 표시하고 강조한다.
- 요청자, 담당자 또는 관리자만 조회할 수 있다.
- 로그인하지 않은 사용자는 로그인 후 원래 주소로 돌아온다.
- 존재하지 않거나 권한이 없는 업무에는 명확한 안내를 표시한다.

이를 위해 백엔드에는 권한 검사를 포함한 단건 조회 API를 추가한다.

```text
GET /payment-tasks/{task_id}
```

## 4. 노션 데이터베이스

`DUP 납부 일정` 데이터베이스를 만들고 노션 캘린더에 연결한다.

| 속성 | 내용 |
| --- | --- |
| 업무명 | `[납부] {요청명}` |
| 납부일 | `due_date` |
| 상태 | `기한 미설정`, `납부 대기`, `기한 초과`, `완료` |
| 담당자 | DUP 담당자 이름 |
| DUP에서 확인 | 납부 업무 바로가기 URL |
| DUP 업무 ID | `PaymentTask.id`, 중복 생성 방지용 |
| 마지막 동기화 | 마지막 정상 동기화 시간 |

노션 일정에는 업무명, 날짜, 상태, 담당자와 DUP 링크만 보낸다. 요청 금액, 설명, PDF, 납부확인증은 DUP에서만 확인한다.

노션 캘린더에서 일정을 선택하면 노션 상세 화면이 열린다. 상세 화면 가장 위에 `DUP에서 확인` 링크를 배치하여 해당 납부 업무로 이동한다.

## 5. 노션 동기화 규칙

동기화 방향은 **DUP → 노션 단방향**으로 한다. 노션에서 수정한 내용은 DUP에 반영하지 않는다.

| DUP 변경 | 노션 처리 |
| --- | --- |
| 납부 요청 생성 | 일정 생성 |
| 업무명 변경 | 일정 제목 변경 |
| 기한 설정·변경 | 일정 날짜 생성·이동 |
| 담당자 변경 | 담당자 변경 |
| 납부 완료 | 삭제하지 않고 상태를 `완료`로 변경 |
| 기한 초과 | 상태를 `기한 초과`로 변경 |

기한이 없는 업무도 노션 데이터베이스에는 생성하지만 날짜가 없으므로 캘린더에는 표시되지 않는다. `기한 미설정` 보기를 별도로 만들고, 기한이 설정되면 자동으로 캘린더에 나타나게 한다.

## 6. 동기화 실패 처리

노션 장애가 DUP의 납부 요청 저장을 막으면 안 된다.

1. DUP 업무를 먼저 저장한다.
2. 노션 동기화를 시도한다.
3. 실패하면 재시도 대상으로 기록한다.
4. 스케줄러가 10분마다 미동기화 업무를 다시 처리한다.
5. `DUP 업무 ID`로 기존 노션 페이지를 확인하여 중복 생성을 막는다.

배포 전 존재하던 미완료 납부 업무는 최초 1회 노션으로 이관한다. 완료 업무는 필요한 기간만 선택하여 이관한다.

## 7. 텔레그램 알림

1차 구현에서는 설정된 텔레그램 채팅방 한 곳으로만 보낸다. 기본 발송 시각은 매일 오전 8시 30분이며 환경설정으로 변경할 수 있게 한다.

```text
🔔 DUP 납부 요약

오늘 납부 3건
기한 초과 1건
기한 미설정 0건

노션 캘린더를 확인해 주세요.
```

업무명, 금액, 계좌번호와 첨부파일은 텔레그램으로 보내지 않는다. 오늘 납부가 0건이어도 정상 작동 확인을 위해 요약 메시지는 보낸다.

집계 기준은 다음과 같다.

- **오늘 납부**: `due_date`가 한국 날짜 기준 오늘이고 `COMPLETED`가 아닌 업무
- **기한 초과**: `due_date`가 오늘보다 이전이고 `COMPLETED`가 아닌 업무
- **기한 미설정**: `due_date`가 없고 `COMPLETED`가 아닌 업무
- **완료 업무**: 모든 알림 숫자에서 제외

## 8. 설정값

비밀값은 프론트엔드에 넣지 않고 백엔드 환경변수로만 관리한다.

```text
NOTION_API_TOKEN=
NOTION_PAYMENT_DATABASE_ID=
FRONTEND_BASE_URL=https://arc.baeksung.kr
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
PAYMENT_SUMMARY_HOUR=8
PAYMENT_SUMMARY_MINUTE=30
```

## 9. 구현 순서

1. 프론트엔드 납부 업무 딥링크와 로그인 후 복귀 구현
2. 백엔드 납부 업무 단건 조회 API 및 권한 검사 구현
3. 노션 데이터베이스와 API 연결
4. 생성·수정·기한 변경·완료 동기화 구현
5. 중복 방지와 실패 재시도 구현
6. 기존 미완료 업무 노션 이관
7. 텔레그램 오전 요약 구현
8. 실제 노션 캘린더와 모바일 텔레그램에서 검증

## 10. 완료 조건

- 납부 요청을 생성하면 노션 일정이 한 번만 생성된다.
- 기한을 바꾸면 기존 노션 일정의 날짜가 바뀐다.
- 완료 처리하면 일정이 삭제되지 않고 `완료`로 표시된다.
- 노션의 `DUP에서 확인` 링크가 정확한 납부 업무를 연다.
- 로그인이 풀린 상태에서도 로그인 후 해당 업무로 돌아온다.
- 권한이 없는 사용자는 링크를 알아도 업무를 볼 수 없다.
- 노션 전송 실패 후 재시도해도 일정이 중복되지 않는다.
- 매일 한국 시간 기준으로 텔레그램 집계가 한 번만 발송된다.
- 오늘 납부, 기한 초과, 기한 미설정 숫자가 DUP 화면과 일치한다.
